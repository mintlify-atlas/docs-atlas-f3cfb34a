---
title: "Quickstart: load, edit, and save a file in tiny-ed"
description: "Walk through a complete tiny-ed session: start the editor, read a file, print and edit lines, append new content, delete a line, then save and quit."
---

This guide walks you through a full editing session with tiny-ed. By the end you will know how to load a file into the buffer, view and modify its contents, and write the result back to disk. The whole workflow takes a few minutes and requires only the compiled binary.

<Steps>
  <Step title="Start the editor">
    Run the tiny-ed binary from the directory where you compiled it. The editor reads commands from standard input and gives no prompt — it is waiting for your first command.

    ```bash
    ./tiny-ed
    ```
  </Step>
  <Step title="Read a file">
    Use `r` followed by a filename to load a file into the buffer. tiny-ed prints the number of lines it read.

    ```text
    r myfile.txt
    12
    ```

    The number `12` means the buffer now contains 12 lines. If the file cannot be opened, tiny-ed prints an error from `perror`.
  </Step>
  <Step title="Print lines">
    Use `p` to print lines. Prefix with a line number to print a specific line, or a range to print multiple consecutive lines.

    ```text
    1p
    Hello, world!
    1,3p
    Hello, world!
    This is line two.
    This is line three.
    ```

    Line addressing starts at 1. If you specify a line number outside the valid range, tiny-ed prints `?`.
  </Step>
  <Step title="Append text">
    Use `[N]a` to append new lines after line N. Type each new line and then enter a lone `.` on its own line to finish input.

    ```text
    2a
    This line goes after line 2.
    And this one after that.
    .
    ```

    After the `.`, tiny-ed returns to command mode. The new lines are now part of the buffer.
  </Step>
  <Step title="Delete a line">
    Use `[N]d` to delete a single line, or `N,Md` to delete a range.

    ```text
    3d
    ```

    This removes line 3 and shifts subsequent lines up. To delete a range:

    ```text
    2,4d
    ```
  </Step>
  <Step title="Save and quit">
    Use `w` to write the buffer to a file. tiny-ed prints the number of bytes written. Then use `q` to exit.

    ```text
    w myfile.txt
    247
    q
    ```
  </Step>
</Steps>

## A complete session

The following shows a full session from start to finish, including starting the editor, loading a file, inspecting and modifying its contents, and saving the result.

```text
$ ./tiny-ed
r notes.txt
5
1,5p
Buy groceries
Call dentist
Fix bug in parser
Write tests
Update README
3d
1,4p
Buy groceries
Call dentist
Write tests
Update README
2a
Schedule follow-up
.
1,5p
Buy groceries
Call dentist
Schedule follow-up
Write tests
Update README
w notes.txt
89
q
```

<Tip>Line addressing starts at 1, not 0. Passing line number 0 or a number greater than the current line count will cause tiny-ed to print `?` and take no action.</Tip>
