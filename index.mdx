tiny-ed is a minimalist line editor built in the spirit of the original Unix `ed`. It reads files into an in-memory buffer and lets you view, add, and delete lines using short, typed commands — no mouse, no menus, just text.

## How tiny-ed works

tiny-ed keeps the entire file in a RAM buffer of up to 1,000 lines. You issue one command at a time — there is no visual display that refreshes. Commands address lines by number, and ranges let you target multiple lines at once.

<Steps>
  <Step title="Build the editor">
    Clone the repo and run `make` to compile the binary.

    ```bash
    git clone https://github.com/BrianDurham/tiny-ed.git
    cd tiny-ed
    make
    ```
  </Step>
  <Step title="Start tiny-ed">
    Run the binary. tiny-ed reads commands from standard input.

    ```bash
    ./tiny-ed
    ```
  </Step>
  <Step title="Load a file">
    Use the `r` command to read a file into the buffer.

    ```
    r notes.txt
    ```

    tiny-ed prints the number of lines read.
  </Step>
  <Step title="Edit and save">
    Print, delete, or append lines, then write the buffer back to disk.

    ```
    1,3p
    2d
    w notes.txt
    q
    ```
  </Step>
</Steps>

<Note>
  tiny-ed is an early-stage learning project. It implements the core `ed` commands but does not aim for full POSIX `ed` compatibility.
</Note>
